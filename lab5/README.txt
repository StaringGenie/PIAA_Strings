>> Алгоритм сборки:
1) Перейти в папку lr5, прописав cd lr5
2) Прописать sudo chmod +x build.sh
3) Затем прописать ./build.sh
4) Для запуска программ перейти в папку build, прописав cd build

>> Алгоритм КМП:
1) Находясь в папке build написать ./SubstrSrch
2) Ввести шаблон для поиска в строке
3) Ввести текст, в котором будет произведен поиск
4) Результатом будут являться индексы вхождений шаблона в текст, если таковые существуют, в противном случае будет выведено "-1"

>> Проверка на циклический сдвиг:
1) Находясь в папке build написать ./CycleShiftSrch
2) Ввести первую строку
3) Ввести вторую строку
4) Результатом будет являться позиция начала второй строки в первой, если такой позиции не существует, будет выведено "-1"

>> Вызов тестирования представленных алгоритмов:
(4 кейса для КМП, 4 кейса для наивного алгоритма поиска, 8 кейсов для циклического сдвига)
1) Находясь в папке build написать ./test

>> Для сравнения эффективности алгоритмов КМП и наивного по количеству операций сравнения:
(данные и строки генерируются случайно, однако вхождения подстроки в строку почти гарантированно будут)
(также, количество генерируемых подстрок и строк, их длину и размер алфавита можно отконфигурировать в define-ах, в коде)
1) Находясь в папке build написать ./CheckOperations

>> Для очистки директории build от всех сгенерированных в процессе сборки файлов, за исключением Makefile:
1) Находясь в папке build написать ./clean
